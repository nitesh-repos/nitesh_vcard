Mailbotize
Python and Selenium

Mailbotize

Python and Selenium

Mailbotize is used in email marketing to enhance the reputation of IP and associated domain of server so as to land sent promotional emails in inbox instead of spam folder.
This project focuses on Not spam technique to warm IP and domain.

First emails are sent to huge pool of pre-owned email accounts of reputed ISP's like Gmail, Hotmail, Yahoo and Aol.
Then system logins to those email accounts through browser and opens spam folder; searches for specific keyword in mails and marks that particular mail as not spam.
Then comes to inbox folder and chooses the same mail which was not spammed in spam folder and performs rest of actions. Like "mark star/important","click on links in mail","reply/forward","archive/delete" etc.

System architecture, Frontend, Framework and Yahoo ISP automation written by me.

System supports following features:

  • Not Spam Machine(NSM)
    Adding and removal of windows machine on the go which opens browser to perform not spam. You can specify number of parallel browsers to open.
  • Proxy support
    Adding authenticated and password less proxy on which to open browser.
  • NSM/Proxy grouping
    You can create proxy/NSM groups and share it to other users.
  • Role based access to portal.
  • Not Spam Profiling.
    You can create various combination of all possible tasks to perform in ISP portal and can specify counts as well as percentage of those tasks and then save it as "NSProfile".
    This helps in reduced amount of time to insert email accounts to perform actions every time. It also helps in keeping track of test cases used.
    You can also share/delete/edit your NSProfile.
  • Compose/ Forward/ Reply mail
  • Click on links in mail.
  • Task scheduling.
    You can specify in which time slot of EST time zone to perform actions.
    How many times to perform actions and delay between every action.
  • Continuous/ Non-Continuous modes.
    In continuous mode once actions are performed on email account; system closes browser, sets next time to perform action again, opens browser after set delayed time and performs action again. This happens in endless manner.
    In non continuous mode the browser is opened only once.
  • Browser profile save.
    If enabled then system saves the opened browser profile. Creates new profile if not exists. This saves time to login email account again as it is already logged in.
  • Control live actions.
    Send "resume/pause/pause_browser/only_login/end" commands to specific email account/proxy browser/ NSM's user.
    Change NSProfile of email accounts.
    Change browser handler if xpath of element changes.
  • Choose browser.
    You can choose between "Chromium/ Chrome/ Firefox" browser to use to perform actions on email account.
  • Proxy checker.
    Check is proxy is black listed.
    Checks if proxy is not connecting.
    Checks if proxy is banned by ISP.
  • Monitor live resource usage
    Shows live CPU and RAM % being used of particular every Not Spam Machine.
  • Proxy report.
    Shows live proxy used/disabled/listed/inactive/clean count.
  • Email account live logs
    Get live logs of email accounts and actions being performed on it. For eg.
    "3 mails marked as not spam"
    "1 mail moved to trash"
    "5 mails read"
    "Opened inbox folder"
    "Password of email account changed successfully"
    "Could not login to account. Reason: Password not working"
  • Report download.
    Download detailed report of Email accounts, Proxy and NSM.

Senderviz

Python and Golang

Senderviz is the software which is capable of sending bulk promotional and transactional emails. Easily bypassing speed of 250k emails/hour from single IP(*conditions apply). It's primarily designed for layman who wants to send bulk emails at very low cost without much of technical knowledge.

90% frontend developed; 10% backend developed

Key features which makes Senderviz standout from rest of the softwares in market:

  • Very easy to use GUI
  • No need to take ssh access to configure rest servers.
  • Centralized portal for all email sending machines.
  • Full control over mail queue.
  • Dynamic template creation for personalized transactional email.
  • Fine tuned job scheduling.
  • Live mail queue management.
  • Live bounce/delivered statistics on portal.
  • Provision to instantly pause mail queue on bounce detection.

Advance features:

  • Customize setting to see report and statistics of your search criteria.
  • Email sent graph of last 59 minutes.
  • Hardbounce, delivered, other errors count live report.
  • Control mail queue by any search criteria.
  • Domain IP group creation. Can group any domain IP of any server together.
  • User role based portal access. Feature can to used to become reseller.
  • Create custom ISP rules and attach to domain IP groups.
  • Create html email template from scratch or copy existing html content to create template.
    Edit in word view or html code editor.
    Use custom variables for dynamic content creation.
  • Adjust sequence of headers/ add custom headers/ remove headers from email draft.
  • Support for text, csv and excel list file upload.
  • Email attachment file upload and feature to use them dynamically in emails.
  • Select any of "html only/ text only/ multi part" type of email to send.
  • Feature to incorporate image in mail itself or to use as hyperlink.
  • Feature to create custom image link or redirection link for opens/clicks tracking.
  • Add/remove MTA server on the go. No need to restart existing MTA servers. That means current mail queue will not restart.
  • Change any ISP settings without affecting current mail queue.
  • Remove or add any domain IP from group without a need to restart MTA.
  • Start/stop/restart/reload/configure/uninstall/clear DNS cache of any MTA server from portal itself.
  • Activate/Deactivate any MTA without affecting rest mail queue.
  • Use previously saved job's settings to send bulk email again.
  • Randomize email list file content or randomize all select files to send emails.
  • Advanced sending pattern:
    Customize mail burst size.
    Send mail to single user from multiple domains(Used for IP warming).
    Select concurrency between "simple round robin" or "multi-threaded".
  • Advanced scheduling:
    Simply start job now.
    Day wise sending. Set start and end time of day.
    Send by hourly limits and specify delays in between.

ISP configuration features:

  • Set all possible limits of no.of mail and recipient.
  • Choose between TLS/ STARTTLS/ none.
  • Choose MX connection type as old or new 8 bit with HELO/EHLO.
  • Enable / Disable auto DKIM signing.
  • Choose which email headers to DKIM sign.
  • TLS certificate support in case of using TLS connection to MX.

Server Panel
Python and Bash scripting

Server Panel

Python and Bash scripting

Server Panel is the Techxyo's email server configuration portal.
Input SSH credentials of any linux server and select among various below mentioned actions to perform:

  • upload
  • configure
  • encrypt_files
  • decrypt_files
  • verify_DNS
  • change_credentials
  • remove
  • clean_server
  • add_to_master
  • kill_task
  • restart_services

You can check the LIVE console log of any server being configured in the portal from SERVER_LOG tab.

The system is designed in a non blocking manner. Thus inserting next server will start it's configuration immediately.

There is custom written wrapper over "paramiko" python library which handles SSH, SFTP and error log management. Plus sending LIVE console log to portal user.

Multiple BASH shell scripts has been written to configure following things on server:
Mysql install and configure with provided DB import and auto user creation.
DNS install and configure with provided Domain-IP resolution.
DKIM private key configuration and adding public keys to DNS records.
PowerMTA configuration and adding private DKIM keys.
Cronjob configure.
Email sending proprietary software configuration.
Adding slave SMTPs to master SMTP.
Apache with SSL configuration.

Technology/languages used:

  • Python for frontend, web sockets and paramiko wrapper
  • Bootstrap 4
  • HTML, Javascript, Jquery
  • Ajax, Json
  • Bash shell scripting.
